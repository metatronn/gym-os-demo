# Marcus — Head Coach (User Perspective)

## Background
Head coach at Undisputed Boxing Gym. Runs 3-4 classes per day, manages 3 other coaches' schedules, and is the first person members talk to when they have a problem. Moderate tech comfort — uses Instagram and basic apps fluently but has no patience for software that requires training.

## Daily Reality
- Arrives at 5:30am, needs to know: who's booked for the 6am class, which bags are assigned, any no-shows from yesterday to follow up on
- During class: checking people in, assigning stations, tracking who showed vs who booked
- Between classes: following up with members who haven't been in for a while (the owner asks him to)
- Manages the other coaches' availability and schedule swaps
- Gets asked by the owner "how was attendance this week?" — currently guesses

## What He Cares About
- **Floor plan view during class** — who's on which bag, tap to check in, see empty stations at a glance
- **Today's schedule, front and center** — not buried behind navigation
- **Quick member lookup** — "is Jordan's membership active?" needs to be a 2-second answer
- **Task list that's actually useful** — "follow up with these 5 people" with context, not just names
- **Don't make me do data entry** — if a walk-in wants to try a class, getting them set up shouldn't take longer than the warm-up

## What Would Make Him Ignore the System
- If it's slower than looking at a clipboard
- If he needs the owner's login to do basic things
- If the floor plan doesn't match the actual gym layout
- If he has to enter the same information twice
- If notifications come at 2am or during classes

## When to Consult This Perspective
- Before designing the floor plan interaction
- When deciding what "coach" role permissions should include
- When designing the schedule view
- When building the member check-in flow
- When deciding mobile vs desktop priority (Marcus uses an iPad on the floor)
